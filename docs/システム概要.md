# システム概要

## ひとことで
**複数の主催者が、自分の Stripe で参加費を集められる「イベント前払い受付」システム**
（個人〜小規模主催者向けの、軽量なイベント決済プラットフォーム／マルチテナント SaaS）。

## 仕組み（3ステップ）
1. 主催者が**招待制**でアカウントを作り、自分の Stripe を連携する（Stripe Connect / OAuth）。
2. イベントを登録すると**申込リンク**が発行され、参加者はそこから
   **事前決済**または**当日支払い**を選んで申し込む。
3. 主催者は参加者名簿を見て、**返金・当日集金チェック・CSV出力**ができる。

## 特徴
- 💰 **入金は各主催者の Stripe 口座へ直接**（当方は資金を仲介せず、現状はプラットフォーム手数料も取らない）。
- 🔒 **カード情報も主催者の秘密鍵も預からない**。決済はすべて Stripe 上、連携は OAuth で
  接続アカウントID（`acct_...`）だけを保持。
- 🗒 **参加者名簿は自前DBに持たない**。名簿は各主催者の Stripe（事前=Checkout セッション、
  当日=課金なしの顧客）から都度取得。DB(SQLite)に保存するのは
  **主催者アカウント・招待・イベント定義**のみ。
- 🎯 狙いは「**ドタキャンでも前払いで取りこぼさない**」こと。

## 支払い方法（イベント単位で設定）
- **事前決済**：申込時に Stripe で前払い。
- **当日支払い**：申込のみ記録し、当日会場で集金（名簿の「集金確認済み」で消し込み）。
- 事前/当日それぞれの**金額**と**受付可否**を個別に設定可能（例：事前 ¥3,000 ／ 当日 ¥4,000）。

## 登場人物
- **プラットフォーム管理者**：招待コードを発行する人（＝このシステムの運営者）。
- **主催者（テナント）**：イベントを作り参加費を集める人。データは主催者ごとに分離。
- **参加者**：主催者から受け取った申込リンクから申し込む人。

## 技術メモ
- PHP 8.1+ / Stripe（Checkout・Connect）/ SQLite（`pdo_sqlite`）。
- 認証はテナントのセッションログイン、サインアップは招待制。
- 詳細な構成・セットアップは [`README.md`](../README.md) を参照。
